# 为什么选择 Rust？

## 系统编程的困境

Rust 是一门系统编程语言，核心目标是：__在没有垃圾回收器的前提下，保证内存安全__。

传统系统语言（C/C++）给程序员极大控制权，但容易引入内存错误（空指针、悬垂指针、缓冲区溢出）。Rust 通过**所有权系统**在编译期检查这些错误，零运行时开销。

## Rust 的答案

Rust 不依赖垃圾回收，也不需要手动管理内存——它通过一套叫做「所有权」的规则，让编译器在你写代码时就找出潜在错误。

# Rust 适合做什么？

## 典型应用场景

Rust 在以下领域表现出色：

- **系统软件**：操作系统内核、设备驱动、嵌入式开发
- **WebAssembly**：在浏览器运行接近原生性能的代码
- **命令行工具**：`ripgrep`、`fd`、`bat` 等知名工具均由 Rust 编写
- **网络服务**：高性能 Web 服务器与异步运行时（Tokio、Actix-web）

## 不适合的场景

Rust 的学习曲线较陡，对于简单脚本或极度注重开发速度的原型阶段，Python、Go 等语言可能是更好的选择。

# 核心特性一览

## 所有权与借用

Rust 的核心创新：变量有明确的「所有者」，引用只能在有效范围内使用，编译器在编译时强制检查。

## 工具链

| 特性 | 说明 |
|------|------|
| 所有权系统 | 编译期内存管理，无 GC，无悬垂指针 |
| 借用检查器 | 防止数据竞争，保证并发安全 |
| 零开销抽象 | 高级语言特性不带运行时代价 |
| 强类型系统 | 代数数据类型、trait、泛型 |
| 优秀工具链 | Cargo 包管理器、rustfmt、Clippy |
