# HaloOS 系列讲解 — 内容编写指南

本文档说明如何在本教程中编写文章、嵌入图片、添加可运行代码和测验题。

---

## 目录结构

```
src/content/chapters/
├── 00-preface/              # 前言（序章，不计入章节编号）
│   └── 00-index.md
├── 01-getting-started/      # 第 1 章
│   ├── 00-index.md          # 章节首页（必须有，文件名以 00- 开头）
│   ├── 01-installation.md   # 第 1 节
│   └── 02-hello-world.md    # 第 2 节
└── 02-ownership/            # 第 2 章
    ├── 00-index.md
    └── 01-ownership-rules.md

public/diagrams/             # SVG 图片
└── rust-compile-flow.svg
```

**命名规则：**
- 文件夹：`NN-章节英文名`，NN 为两位数字
- 文章文件：`NN-文章英文名.md`，`00-` 开头的是章节首页
- 文件名的字母顺序决定侧边栏排列顺序
- 练习文章建议命名为 `NN-practice.md`

## 文章 Frontmatter

每篇文章必须以以下 frontmatter 开头：

```yaml
---
title: "文章标题"
description: "一句话简介，显示在课程目录里"
difficulty: beginner        # beginner / intermediate / advanced
estimatedTime: 15           # 预计阅读分钟数（整数）
keywords: ["关键词1", "关键词2"]
---
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | 字符串 | 文章标题，显示在侧边栏和页面顶部 |
| `description` | 字符串 | 简介，显示在首页课程目录 |
| `difficulty` | 枚举 | 只能是 `beginner` / `intermediate` / `advanced` |
| `estimatedTime` | 数字 | 预计学习时间（分钟） |
| `keywords` | 数组 | 关键词标签，显示在课程目录 |

## 正文排版

使用标准 Markdown 语法。`## 二级标题` 会自动出现在右侧目录（TOC），进度系统按二级标题拆分节段。

```markdown
# 一级标题（页面主标题）

## 二级标题（会出现在右侧目录，进度按此拆分）

### 三级标题

正文内容，支持 **加粗**、*斜体*、`行内代码`。

> 引用块用于提示或注意事项。
```

## 嵌入图片（亿图 SVG）

**工作流：**
1. 在亿图中绘制好图，导出时勾选「文字转曲线」
2. 将 SVG 文件放到 `public/diagrams/` 目录
3. 在文章中用 `<img>` 标签引用

```html
<img src="/RustCourse/diagrams/文件名.svg" alt="图片说明" style="max-width:100%;border-radius:8px;" />
```

**注意：** 路径必须以 `/RustCourse/diagrams/` 开头（GitHub Pages 子路径）。本地开发也可以正常访问。

## 可运行代码块

读者可直接在页面上点击「运行」执行代码，调用 Rust Playground API。

**基本用法：**

````markdown
```rust runnable
fn main() {
    println!("Hello, Rust!");
}
```
````

**隐藏样板代码（# 开头的行）：**

页面上只显示有意义的核心代码，`# ` 开头的行在渲染时隐藏，但实际执行时包含。

````markdown
```rust runnable
# fn main() {
    let s = String::from("hello");
    println!("{}", s);
# }
```
````

**演示编译错误（expect-error）：**

代码会显示编译错误信息，用于讲解 Rust 的错误提示机制。

````markdown
```rust runnable expect-error
# fn main() {
    let s1 = String::from("hello");
    let s2 = s1;
    println!("{}", s1); // s1 已失效，这是编译错误
# }
```
````

## 可编辑代码练习

读者可以在页面上直接编辑代码并运行，用于编程练习题。

**基本格式：**

````markdown
```rust editable
fn main() {
    // TODO: 在此处补全代码
}
```
````

**带预期输出验证（编辑框下方紧跟 expected 块）：**

读者提交后会与预期输出对比，自动判断是否正确，并记录到学习进度。

````markdown
```rust editable
fn main() {
    // TODO: 打印 Hello, World!
}
```

```expected
Hello, World!
```
````

## 测验题

### 单选题

````markdown
```quiz single
Q: 问题文本
- 错误选项 A
- 错误选项 B
+ 正确选项（前面加 +）
- 错误选项 D
E: 解析说明，提交后显示给读者。
```
````

### 多选题

````markdown
```quiz multi
Q: 下列哪些是 Cargo 的功能？
+ 创建新项目（cargo new）
+ 构建项目（cargo build）
- 安装操作系统补丁
+ 运行测试（cargo test）
E: Cargo 负责项目创建、依赖管理、构建、测试等，不管系统级操作。
```
````

**语法规则：**

| 标记 | 含义 |
|------|------|
| `Q:` | 题目（必须，只有一行） |
| `-` | 错误选项 |
| `+` | 正确选项（单选只能有一个 `+`，多选可以有多个） |
| `E:` | 解析说明（必须，提交后显示） |

## 常见问题

**Q: 新建了文章但侧边栏没出现？**
重启 dev server（`npm run dev`），Astro 的内容集合需要重新同步。

**Q: 图片路径对了但图裂了？**
检查 `public/diagrams/` 下文件名是否一致（区分大小写），路径必须是 `/RustCourse/diagrams/文件名.svg`。

**Q: 测验题提交后没有进度记录？**
确认 `expected` 块（编程题）或 `E:` 解析（选择题）没有缺失，语法不完整时组件会静默失败。

**Q: 证书何时开放？**
`src/pages/certificate.astro` 顶部有一个开关：
```typescript
const CERTIFICATE_ENABLED = false; // 改为 true 即可开放
```
