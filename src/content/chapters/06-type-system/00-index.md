# 章节概览

你已经学过 Rust 的基本类型（整数、浮点、布尔、字符等）。本章深入探讨 **Rust 类型系统** 的三个核心主题：

## 1. **类型推导**（`01-type-inference.md`）

Rust 的编译器很"聪明"。它不仅在初始化时推导类型，还会根据变量的**使用方式**来推断类型。理解推导规则能让你写出更简洁的代码。

**核心概念：**
- 从初始值推导
- 从使用方式推导
- 何时必须显式标注
- 类型推导的限制

## 2. **类型铸造**（`02-type-casting.md`）

当需要在不同类型间进行**显式转换**时，使用 `as` 关键字。这是控制类型转换的低级工具，需要理解整数溢出、浮点精度等细节。

**核心概念：**
- 使用 `as` 进行显式转换
- 整数转换规则与二进制补码
- 浮点数转换的精度问题
- 常见的转换陷阱

## 3. **类型别名**（`03-type-aliases.md`）

使用 `type` 关键字为复杂类型起别名，提高代码可读性。这是一个简单但强大的工具，特别是对于泛型重复类型。

**核心概念：**
- `type` 语法和命名规范
- 别名 vs 新类型的区别
- 何时使用别名的最佳实践

## 4. **转换 Trait**（`04-conversions.md`）

对于**自定义类型的转换**，Rust 提供了一套优雅的 trait 系统：

- **`From<T>` / `Into<T>`**：总是成功的转换
- **`TryFrom<T>` / `TryInto<T>`**：可能失败的转换（返回 Result）
- **`ToString` / `FromStr`**：字符串的特殊转换

**核心概念：**
- 为自定义类型实现转换 trait
- 理解 From 和 Into 的互补关系
- 使用 TryFrom 处理可能失败的转换
- 字符串与其他类型的互转

## 学习路径

**建议顺序：**
1. 先学 **类型推导** — 理解编译器的"聪明"
2. 再学 **类型铸造** — 理解显式转换的细节
3. 然后 **类型别名** — 简单但有用的工具
4. 最后 **转换 Trait** — 强大而优雅的解决方案

每篇都包含测验题和编程练习，确保深入理解。
