# 什么是所有权

## 内存管理的挑战

所有权是 Rust 最独特的特性，它让 Rust 在不需要垃圾回收器的情况下保证内存安全。

传统语言要么使用 GC 自动回收（有运行时开销），要么手动管理内存（容易出错）。Rust 选择了第三条路：编译期所有权检查。

## 作用域与自动回收

当变量离开作用域时，Rust 自动调用 `drop` 函数释放内存，无需手动处理。

# 所有权规则

## 三条核心规则

Rust 中每一个值都遵循以下规则：

1. 每个值都有一个 **所有者**（owner）
2. 值在任一时刻只能有 **一个** 所有者
3. 当所有者离开作用域，值将被自动丢弃（drop）

## 规则的意义

这三条规则共同保证：同一块内存不会被两个地方同时释放（double free），也不会在释放后再次使用（use-after-free）。
