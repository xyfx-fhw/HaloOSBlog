# 章节概览

你已经掌握了 Rust 的基础类型和自定义类型。现在是时候学习 Rust 标准库中最常用的**集合类型**了。

在实际开发中，你很少使用单个值。大多数时候，你需要管理一组数据。本章介绍了 Rust 标准库中三种最重要的集合类型，它们几乎出现在每个 Rust 程序中：

1. **Vec\<T\>** — 可动态增长的数组，用于存储一系列相同类型的值
2. **String** — 可变字符串，掌握 String 与 &str 的区别
3. **HashMap\<K, V\>** — 键值对集合，通过键快速查找值

## 为什么需要这三种类型

| 类型 | 用途 | 特点 |
|------|------|------|
| **Vec\<T\>** | 存储多个值 | 按位置访问，支持动态增长 |
| **String** | 处理文本 | 可修改，UTF-8 编码，拥有所有权 |
| **HashMap** | 关联查找 | 通过键快速查找，无序 |

## 学习路线

每个集合类型都围绕一个核心主题：**所有权**。

- **Vec\<T\>** 讲授向量如何拥有元素、借用规则如何影响迭代
- **String** 深入 Rust 为什么有两种字符串类型，以及 UTF-8 编码的复杂性
- **HashMap** 展示如何处理键和值的所有权转移

通过这一章，你将深刻理解 Rust 的所有权系统如何在集合上发挥作用。
