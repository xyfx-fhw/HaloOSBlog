## 这一章讲什么

程序出 bug 是家常便饭。新手常见的第一反应是疯狂插入 `println!`——这能解决问题，但 Rust 提供了更好的工具。

这一章介绍四种互补的调试手段：

| 文章 | 适合什么情况 |
|------|------------|
| **dbg! 宏** | 临时查看某个表达式的值，快速定位逻辑错误 |
| **IDE 调试器** | 需要单步执行、观察多个变量状态的复杂 bug |
| **日志输出** | 长期运行的程序、库代码，需要可控的诊断信息 |

## 学完能做什么

- 用 `dbg!` 快速打印调试，不破坏表达式逻辑
- 在 VS Code 中配置断点、单步执行 Rust 程序
- 用 `log` + `env_logger` 给程序加上可过滤的日志

> 调试能力是工程师的核心素养。掌握这些工具，遇到 bug 不再靠"感觉"，而是靠系统化排查。
