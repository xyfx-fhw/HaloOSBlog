# 章节概览

你已经学过 Rust 的基本类型（整数、浮点、布尔、字符串等）。现在是时候创建自己的类型了。

Rust 提供了强大的类型系统，让你能够以安全、表达性强的方式组织和处理数据。本章将介绍：

1. **结构体** — 将相关的数据组织在一起
2. **方法与关联函数** — 给数据类型附加行为
3. **枚举** — 表示一个值是某个离散集合中的一种
4. **Option<T>** — 安全地处理值的存在或缺失
5. **模式匹配** — 强大的控制流工具
6. **常量与静态变量** — 编译期已知的值

# 为什么需要自定义类型

假设你要写一个图形库，需要表示点的坐标。用基本类型：

```rust runnable
fn main() {
    // 用三个独立的变量（混乱且容易出错）
    let point_x = 10;
    let point_y = 20;
    let point_z = 30;
    
    // 很难清楚地表达这些数据是相关的
    println!("点：({}, {}, {})", point_x, point_y, point_z);
}
```

用自定义类型（结构体）：

```rust runnable
struct Point {
    x: i32,
    y: i32,
    z: i32,
}

fn main() {
    let point = Point { x: 10, y: 20, z: 30 };
    println!("点：({}, {}, {})", point.x, point.y, point.z);
}
```

第二种方式清晰、类型安全、易于维护。

# 学习路径建议

- 从**结构体**开始，理解如何组织数据
- 学**方法与关联函数**，给数据添加行为
- 理解**枚举**，学会用类型表示离散的选择
- 掌握 **Option<T>**，安全处理可选值
- 学 **match 和 if let**，在不同情况下执行不同代码
- 了解**常量与静态变量**，定义编译期已知的值
- 最后用**综合练习**巩固所有概念

# 小贴士

1. **优先用 `struct` 还是 `enum`？**
   - 结构体：当需要将多个相关数据组合在一起时
   - 枚举：当一个值只能是多个选项之一时

2. **何时用方法 vs 关联函数？**
   - 方法：作用于实例的行为（第一参数是 `self`）
   - 关联函数：不作用于实例的功能（通常是构造函数）

3. **Option vs 错误处理**
   - `Option<T>` —— 表示值可能存在或不存在
   - `Result<T, E>` —— 表示操作成功或失败（后续章节讲解）

4. **match 的穷尽性很强大**
   - 编译器强制处理所有情况，防止逻辑错误
   - 当枚举添加新成员时，编译器会警告你更新 match
