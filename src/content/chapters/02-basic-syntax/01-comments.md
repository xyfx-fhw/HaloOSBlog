# 注释语法

注释不参与程序运行，却是代码不可缺少的一部分。Rust 有四种注释形式，每种都有不同的用途和使用场景。

## 行注释 `//`

行注释是最常见的形式，`//` 后面到行尾的所有内容都会被编译器忽略：

```rust runnable
fn main() {
    // 这是一行注释
    println!("Hello"); // 也可以写在代码行末尾

    // 注释掉的代码不会执行：
    // println!("这行不会输出");
}
```

Rust 社区的惯例是**优先使用行注释**，而不是块注释。行注释更清晰、更容易追踪每行的意图。

## 块注释 `/* */`

块注释可以跨越多行，常用于临时注释掉一大段代码：

```rust runnable
fn main() {
    /*
     * 这是块注释。
     * 缩进和星号只是风格，不是语法要求。
     */

    let x = 5 + /* 90 + */ 5; // 块注释可以嵌入表达式中间！
    println!("x = {}", x);     // 输出 10，不是 100
}
```

Rust 的块注释有一个独特之处：**支持嵌套**。

```rust runnable
fn main() {
    /* 外层注释
        /* 内层注释也可以 */
    还在外层注释中 */
    println!("块注释可以嵌套");
}
```

> 嵌套块注释在 C 语言中是不合法的，但 Rust 支持。这让你可以用 `/* */` 快速注释掉已经包含块注释的代码块。

## 文档注释 `///`

`///` 用于为**紧跟在它后面的代码项**（函数、结构体、模块等）生成 HTML 格式的 API 文档，内容支持 Markdown：

```rust runnable
/// 计算两个整数的和。
///
/// # 示例
///
/// ```
/// let result = add(2, 3);
/// assert_eq!(result, 5);
/// ```
fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn main() {
    println!("{}", add(2, 3));
}
```

运行 `cargo doc --open` 后，`///` 的内容会渲染成漂亮的网页文档——这是 Rust 生态的标准文档格式，所有开源 crate 都遵循这个惯例。

## 内部文档注释 `//!`

`//!` 与 `///` 方向相反——它为**包含它的项**（通常是文件顶部）生成文档，常用于描述整个模块或 crate：

```rust runnable
//! 这是当前模块的描述。
//! 一般放在文件最顶部，用于说明模块的整体用途。

fn main() {
    println!("模块文档注释示例");
}
```

简单记：`///` 是"我在描述下面的东西"，`//!` 是"我在描述我所在的容器"。

> 文档注释和内部文档注释的完整用法（Markdown 语法、示例测试、cargo doc 工作流）在[项目工程化：文档注释与 doctest](/RustCourse/chapters/08-engineering/03-doc-comments) 中专门讲解，当前了解即可。

## 四种注释速查

| 形式 | 用途 | 生成文档 |
|------|------|---------|
| `//` | 普通行注释 | 否 |
| `/* */` | 普通块注释，可嵌套 | 否 |
| `///` | 为下方的项生成文档 | **是** |
| `//!` | 为所在模块/crate 生成文档 | **是** |

# 练习题

## 注释的执行

```rust
fn main() {
    // println!("A");
    println!("B");
    /* println!("C"); */
}
```

```quiz single
Q: 运行上面的代码，输出是什么？
+ B
- A
- B 和 C
- A 和 B
E: `//` 和 `/* */` 内的代码都会被编译器忽略，只有 `println!("B")` 会执行，输出 B。
```

## 块注释的特性

```quiz single
Q: Rust 的块注释 `/* */` 与 C 语言的块注释相比，有什么独特之处？
- 块注释不能跨越多行
+ 块注释支持嵌套，即 `/* /* 内层 */ */` 是合法的
- 块注释只能写在单独的行，不能嵌入表达式中间
- 块注释会生成文档
E: Rust 的块注释支持嵌套，而 C 语言不支持。这让你可以用 `/* */` 注释掉已经含有块注释的代码段。块注释也可以嵌入表达式中间，如 `let x = 5 + /* 90 + */ 5;`。
```

## 文档注释的方向

```quiz single
Q: `///` 和 `//!` 的核心区别是什么？
- `///` 生成文档，`//!` 不生成文档
+ `///` 描述紧跟在它后面的项，`//!` 描述包含它的容器（如模块）
- `///` 用于函数，`//!` 用于结构体
- 两者完全相同，只是风格不同
E: `///` 是"向外"注释，描述下方的函数、结构体等；`//!` 是"向内"注释，描述它所在的模块或 crate。`//!` 通常出现在文件顶部。
```

## 哪些注释会生成 API 文档

```quiz multi
Q: 运行 `cargo doc` 时，哪些注释的内容会出现在生成的 HTML 文档中？
- `/* 块注释 */`
+ `//! 内部文档注释`
- `// 普通行注释`
+ `/// 外部文档注释`
E: 只有 `///` 和 `//!` 是文档注释，会被 `cargo doc` 解析为 API 文档。`//` 和 `/* */` 是普通注释，编译器完全忽略，不会出现在文档里。
```

## 推荐的注释风格

```quiz single
Q: Rust 社区对于普通注释，更推荐哪种形式？
+ 行注释 `//`
- 两者完全等价，没有推荐
- 文档注释 `///`
- 块注释 `/* */`
E: Rust 社区惯例是优先使用行注释 `//`，块注释主要用于临时注释掉大段代码。rustfmt 也默认按照行注释风格处理。文档注释 `///` 有特殊用途，不用于普通注释。
```
